Open Generate Signed Dialog

In Android Studio:

Build → Generate Signed Bundle / APK

Select Format

Choose your preferred format:

Select: Android App Bundle (AAB)

Note: AAB is required for Play Store after August 2021

Create or Select Keystore

For new keystore, click "Create new..." and fill:

Key store path: [Choose secure location] Password: [Strong password] Confirm: [Re-enter] Alias: [e.g., notesreminder_key] Password: [Different password] Confirm: [Re-enter] Validity: [25 years] Certificate Details: First/Last Name: [Your Name] Organizational Unit: [Development] Organization: [Company Name] City/Locality: [Your City] State/Province: [Your State] Country Code: [e.g., US, IN]

Configure Build

Build Variants: release Signature Versions: ☑ V1 (Jar Signature) ☑ V2 (Full APK Signature) ☑ V3 (v3 Signature Scheme) Destination folder: [Choose output location]

Tips

• V1 signature is for older Android devices

• V2/V3 provide better security and faster installation

Finish and Wait

Click Finish and wait 2-5 minutes for build completion.

$ BUILD SUCCESSFUL in 3m 12s 35 actionable tasks: 35 executed Output: app/build/outputs/bundle/release/app-release.aab

Generate Keystore via Terminal

Navigate to your project directory:

cd /path/to/your/android/project

Generate keystore:

keytool -genkey -v -keystore notesreminder.jks \ -alias notesreminder_key \ -keyalg RSA -keysize 2048 -validity 10000
Enter keystore password: Re-enter new password: What is your first and last name? [Unknown]: John Developer What is the name of your organizational unit? [Unknown]: Development ... (fill all details) Is CN=John Developer, OU=Development... correct? [no]: yes

Configure build.gradle

Edit app/build.gradle:

android { signingConfigs { release { storeFile file("notesreminder.jks") storePassword "your_password" keyAlias "notesreminder_key" keyPassword "your_key_password" } } buildTypes { release { signingConfig signingConfigs.release minifyEnabled false shrinkResources false } } }

Generate Build

For AAB (recommended):

./gradlew bundleRelease

For APK:

./gradlew assembleRelease

Output Locations

AAB: app/build/outputs/bundle/release/app-release.aab

APK: app/build/outputs/apk/release/app-release.apk

Create Secure Properties File

Create or edit ~/.gradle/gradle.properties:

# DO NOT COMMIT THIS FILE TO VERSION CONTROL! RELEASE_STORE_FILE=notesreminder.jks RELEASE_STORE_PASSWORD=your_secure_password RELEASE_KEY_ALIAS=notesreminder_key RELEASE_KEY_PASSWORD=your_key_password

Update build.gradle

android { signingConfigs { release { storeFile file(RELEASE_STORE_FILE) storePassword RELEASE_STORE_PASSWORD keyAlias RELEASE_KEY_ALIAS keyPassword RELEASE_KEY_PASSWORD } } buildTypes { release { signingConfig signingConfigs.release minifyEnabled true shrinkResources true proguardFiles getDefaultProguardFile( 'proguard-android-optimize.txt'), 'proguard-rules.pro' } } }

Add to .gitignore

Ensure security by adding to .gitignore:

# Keystore files *.keystore *.jks # Gradle properties gradle.properties *.properties # Keys directory keys/ secure/

Common Errors & Solutions

Error: Keystore was tampered with

Solution: Verify passwords are correct. Check if using correct keystore file.

Error: Failed to read key from keystore

Solution: Check alias spelling. Verify alias exists:

keytool -list -v -keystore your.keystore

Build takes too long

Solution: First build takes longer. Increase Gradle heap size in gradle.properties:

org.gradle.jvmargs=-Xmx4096m -XX:MaxPermSize=512m

APK/AAB size too large

Solution: Enable optimization:

minifyEnabled true shrinkResources true enableR8 = true

Verify Your Build

Check APK signing:

# For APK keytool -printcert -jarfile app-release.apk # For AAB (extract first) bundletool dump manifest --bundle=app-release.aab

Check APK details:

aapt dump badging app-release.apk | findstr version

Test Release Build

# Install on connected device adb install app-release.apk # Install app bundle (convert to APKs first) bundletool build-apks --bundle=app-release.aab \ --output=app-release.apks \ --ks=your.keystore --ks-pass=pass:your_password bundletool install-apks --apks=app-release.apks

Upload Steps

  1. Go to Google Play Console
  2. Select your app → Production → Create new release
  3. Upload your AAB file
  4. Fill in release notes (what's new)
  5. Select countries for distribution
  6. Review and publish
  7. Wait for review (2-48 hours)

Release Strategy

Recommended rollout:

1. Internal testing (team only)

2. Closed testing (selected users)

3. Open testing (public testing)

4. Production (full release)
